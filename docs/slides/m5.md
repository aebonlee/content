## 세션 1 — 자연어 UI 시안 생성

Stitch는 **자연어 프롬프트 → UI 시안** 생성 도구.
"무엇을 위한 화면인지"를 명확히 주는 게 핵심.

---

## 핵심 개념 ①

- **Google Stitch** — 자연어로 앱/웹 UI 시안을 생성하는 구글 도구. 화면 목적·섹션·톤을 주면 시안과 레이아웃을 만든다.
- **디자인 브리프** — 목적·타깃·핵심 행동(CTA)·톤·필수 섹션을 한 문단으로. **시안 품질은 브리프 품질에 비례**.
- **반복(Iteration)** — 한 번에 완성하지 않고 "히어로를 더 크게", "후기 섹션 추가"처럼 부분 수정으로 다듬는다.

---

## 실습 절차 — 첫 시안

1. 페이지 목적과 핵심 행동(CTA)을 한 문장으로 정한다.
2. 필수 섹션 목록을 적는다(히어로·특징·후기·CTA 등).
3. Stitch에 디자인 브리프를 입력해 첫 시안을 생성한다.
4. 부분 수정 지시로 2~3회 반복해 다듬는다.

---

## 실습 프롬프트 — Stitch 디자인 브리프

```프롬프트
Create a landing page for a craft non-alcoholic beer brand.
Goal: get visitors to pre-order.
Tone: warm, premium, minimal (amber + navy).
Sections: hero with one-line headline + CTA button,
3 feature cards, customer quotes, FAQ, final CTA.
Mobile-first layout.
```

> 🗣️ 흔한 실수: 목적/CTA 없이 "예쁘게"만 요청하면 방향 없는 시안. 한 번에 완벽 기대 금지 → 부분 수정 반복이 정석.

📦 결과물: **홍보용 페이지 UI 시안 1개**(히어로·특징·CTA 포함)

## 세션 2 — 페이지 구조 설계

랜딩/이벤트 페이지는 **"한 페이지 = 하나의 목표"**로 화면 흐름을 설계한다.

---

## 핵심 개념 ②

- **랜딩페이지 1목표 원칙** — 한 페이지는 하나의 전환 목표(구매·신청·문의)만. 목표가 여러 개면 전환율이 떨어진다.
- **시각적 흐름(F/Z 패턴)** — 시선 동선을 고려해 핵심 메시지와 CTA를 동선 위에 배치.
- **와이어프레임** — 색·이미지 전에 박스로 구조(무엇이 위에/아래에)를 먼저 잡는 저해상도 설계.

---

## 실습 절차 — 구조 잡기

1. 페이지의 단일 목표를 적는다.
2. 섹션 순서를 와이어프레임(박스)으로 배치한다.
3. Stitch 시안을 이 구조에 맞춰 조정한다.
4. CTA가 첫 화면과 마지막에 반복되는지 확인한다.

---

## 실습 프롬프트 — 구조 조정 지시

```프롬프트
Refine the page: move the primary CTA into the hero,
add a sticky top bar with the CTA,
reduce sections to: hero, 3 benefits, social proof, CTA.
Keep it single-goal: pre-order.
```

---

## 흔한 실수

- ⚠️ 한 페이지에 목표가 여럿이면 전환이 분산된다 → **1페이지 1목표**.
- ⚠️ CTA가 한 번만 있으면 놓친다 → **상단·하단 반복 배치**.

📦 결과물: **캠페인용 페이지 구조(와이어프레임) + 조정된 Stitch 시안**
